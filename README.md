# MustafaAlshehab.github.io

[![Deploy to Github Pages](https://github.com/MustafaAlshehab/MustafaAlshehab.github.io/actions/workflows/deploy-site.yaml/badge.svg?branch=develop)](https://github.com/MustafaAlshehab/MustafaAlshehab.github.io/actions/workflows/deploy-site.yaml)

Personal portfolio site built with [Hugo](https://gohugo.io/) and the [Toha](https://github.com/hugo-toha/toha) theme. Live at [mustafaalshehab.github.io](https://mustafaalshehab.github.io).

## Branch model

| Branch | Purpose |
|--------|---------|
| `develop` | Source: Hugo config, content, assets, and CI workflow |
| `master` | Published static site (`public/`) deployed by GitHub Actions |

Pushes to `develop` trigger a build and deploy the output to `master` for GitHub Pages.

## Version lock

| Tool | Version |
|------|---------|
| Hugo | **0.164.0** (extended) |
| Toha | **v4.16.0** (Hugo Module) |
| Go | ≥ 1.21 |
| Node.js | ≥ 18 (20 recommended) |

Toha v4.16.0 requires Hugo ≥ 0.163.0. When bumping the theme, upgrade Hugo (local + CI) in lockstep.

## Prerequisites

- [Hugo extended](https://gohugo.io/installation/) 0.164.0
- [Go](https://go.dev/dl/) 1.21+
- [Node.js](https://nodejs.org/) 18+ and npm 8+

Verify:

```bash
hugo version   # should show v0.164.0+extended
go version
node -v
```

## Local setup

```bash
git clone https://github.com/MustafaAlshehab/MustafaAlshehab.github.io.git
cd MustafaAlshehab.github.io
git checkout develop

# Download the Toha theme module
hugo mod tidy

# Generate and install Node dependencies used by the theme
hugo mod npm pack
npm install

# Run the local preview server
hugo server -w
```

Open [http://localhost:1313](http://localhost:1313).

## Build

Production build (writes to `public/`):

```bash
hugo --minify
```

## Project layout

```
.
├── assets/images/     # Logos, background, profile, skill/project icons
├── config.yaml        # Site config + Hugo module mounts
├── data/en/           # Homepage sections and author info (YAML)
├── static/files/      # Static downloads (e.g. resume.pdf)
├── go.mod / go.sum    # Hugo module lock (Toha pin)
├── package.json       # Generated by `hugo mod npm pack`
└── .github/workflows/ # Deploy to GitHub Pages
```

Homepage sections are driven by files under `data/en/`:

- `author.yaml` — name, photo, contact, hero summary lines
- `site.yaml` — copyright and meta description
- `sections/about.yaml`, `skills.yaml`, `experiences.yaml`, `education.yaml`, `projects.yaml`, …

Images referenced in config and data should live under `assets/images/` and use paths like `/images/...` so Hugo Pipes can process them.

## Editing content

1. Update the relevant YAML under `data/en/`.
2. Add or replace images in `assets/images/` as needed.
3. Put downloadable files (resume, etc.) in `static/files/` and link them via `resourceLinks` in `about.yaml`.
4. Preview with `hugo server -w`, then commit and push to `develop`.

## Updating the Toha theme

```bash
# Pin a specific release (recommended)
hugo mod get github.com/hugo-toha/toha/v4@v4.16.0
hugo mod tidy
hugo mod npm pack
npm install
hugo --minify
```

When moving to a newer Toha release, check its minimum Hugo version and upgrade local Hugo plus the `hugo-version` in `.github/workflows/deploy-site.yaml` together.

## CI / deploy

Workflow: [`.github/workflows/deploy-site.yaml`](.github/workflows/deploy-site.yaml)

On each push to `develop` it:

1. Installs Go, Hugo 0.164.0 extended, and Node 20
2. Runs `hugo mod tidy`, `hugo mod npm pack`, and `npm ci`
3. Builds with `hugo --minify`
4. Publishes `public/` to the `master` branch (GitHub Pages + CNAME)
